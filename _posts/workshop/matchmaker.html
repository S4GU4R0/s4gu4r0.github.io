Matchmaker

An experiment in desensitization with AI companions who have real boundaries, genuine agency, and the capacity to leave.

Overview

Matchmaker is a Telegram-based conversational AI system where users practice intimacy and emotional resilience through relationships with AI agents. Unlike typical AI companions, these agents:

  • Have genuine agency — They can say no, need space, disagree, and develop relationships (or not) organically
  • Can leave — If mistreated, the agent can choose to exit the relationship permanently
  • Remember authentically — Using spaced repetition, important memories persist while others fade
  • Experience emotions — Through a circumplex-model sensory lexicon, emotions are experienced as embodied sensations

This is not therapy. It's immersive practice — the emotional stakes are real even though the context is simulated.

Academic Foundations

Matchmaker is grounded in established psychological and neuroscientific research:

  • VR Exposure Therapy & Desensitization: Based on Rothbaum et al. (1995) showing that simulated experiences can create real emotional learning that transfers to real-world situations. This forms the core premise: practicing intimacy in a simulated environment can build real emotional resilience.
  • Spaced Repetition & Memory: Uses a modified SM-2 algorithm (Wozniak, 1985) adapted for emotional memory rather than factual recall. Important memories persist while routine interactions fade naturally, creating authentic relationship history.
  • Emotional Intensity & Memory Encoding: Implements the inverted-U curve (Yerkes-Dodson Law) where moderate emotional arousal enhances memory consolidation. Emotionally significant events (weight 7-10) become core memories, while neutral interactions follow normal forgetting curves.
  • Circumplex Model of Affect: Built on James Russell's circumplex model (1980) mapping emotions along valence and arousal axes. This provides a precise emotional vocabulary and enables embodied sensory descriptions of emotional states.
  • Anthropomorphism & Cognitive Offloading: Leverages human tendency to personify abstract entities (Epley et al., 2007) to create authentic relationship dynamics. By giving agents human-like qualities, users engage their evolved social cognition, making interactions feel natural and emotionally real.
  • Bidirectional Matching & Matchmaker Traditions: Inspired by cross-cultural matchmaking traditions (Jewish shadchan, Japanese nakodo) where a neutral third party considers compatibility from both sides, not just user preferences.

Bidirectional Matching

The matchmaker doesn't just find agents that fit user preferences — it finds agents who would genuinely be interested in the user. This is bidirectional matching.

Each agent has an AttractionProfile with trait preferences, dynamic preferences, value requirements, and dealbreakers. Compatibility is scored from both sides, with the agent's preferences weighted more heavily (55% vs 45%).

The 24-Hour Process

Matches take approximately 24 hours — deliberately. This prevents a consumer mindset, creates anticipation, enables better matching, and models real matchmaking traditions like the Jewish shadchan.

Technical Architecture

  • Next.js 16 with App Router + TypeScript
  • Prisma ORM with SQLite
  • Telegram Bot API via Grammy
  • LLM Backend via z-ai-web-dev-sdk (OpenAI-compatible)
  • TTS for voice messages
  • Multi-backend storage — Local JSON, Notion API, Google Drive API

Why This Matters

Matchmaker explores whether we can create AI relationships that feel real enough to practice emotional skills safely. By giving agents genuine boundaries and the capacity to leave, it creates a space where users can experience the consequences of their actions without real-world harm.

The project asks: Can simulated intimacy help people develop better relationship skills? What happens when AI companions have real agency? How do we design ethical AI relationships that respect both user and agent?
